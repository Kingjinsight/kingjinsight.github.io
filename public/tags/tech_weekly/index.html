Reduction is like to prove a known NP-complete question, and transfer it into a NP question X, then X is also a NP-complete question MCP(Model context protocol) can be understand as a uniform langauge between LLM and external services. Just like a translator, to let different AI application(llm, cursor) and different tools(github, calendar) to talk easily, rather than develop a new API between them. In the past, every time you want an AI assistant to interact with a new tool, you would need to develop a separate interface for each tool. This led to a huge amount of work because you’re have to create multiple integrations. MCP simplifies this by ensuring that all AI applications only need to support MCp, and all tools only need to support MCP, allowing them to communicate with each other easily and reducing development cost.
